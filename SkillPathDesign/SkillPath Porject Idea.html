04 手机端友好设计 Mobile-First Responsive

为什么不做原生 App？

  • 技术 ROI 最优：响应式 Web 工作量 +20%，覆盖率 +80%
  • 部署最简单：无需 App Store 审核，改了立刻生效
  • 朋友愿意试：一个 URL 发过去就能用，没下载门槛
  • 面试 Demo 方便：面试官手机扫码就能体验
  • 核心技术点：Tailwind 断点、触摸手势、虚拟键盘适配、响应式图表
05 成本控制系统 Because Real Users Cost Money
🛡️
四层防护架构
Protect your OpenAI credits
L1 · IP + 邮箱速率限制
注册接口：同 IP 每天最多 3 个账号，防止批量注册
express-rate-limit · Redis counter
L2 · 用户 Token 配额
免费用户每天 50K tokens（约 5 次路径生成），超出显示"明日再试"
MongoDB daily_usage collection
L3 · 模型分级调用
简单任务用 GPT-4o-mini（$0.15/M），复杂规划才用 GPT-4o（$2.5/M）
动态 model router
L4 · 全局熔断
每日 OpenAI 总花费 >$2 自动关服务并邮件告警，避免失控
Cron + SendGrid
💡 预算目标：支持 100+ 注册用户、50 日活，OpenAI 月成本控制在 <$30（配合 OpenAI 免费额度 + 分级调用，大概率能压到 $15 以内）
06 Agent 执行流程 LangGraph Runtime
1
用户输入学习请求
"我想 14 天内搞懂 React Hooks，每天 1 小时"
Input → StateGraph.START · 配额检查
2
Planner Node —— 任务规划
GPT-4o 分解为学习节点，Zod 验证结构化输出
LLM + structured output · ~2000 tokens
3
Researcher Node —— 资源搜索
并行调用 Tavily + YouTube，LLM 评分筛选 Top 3
Parallel tool calls · ranking
4
Memory 检索 —— 个性化调整
从 Pinecone 查询用户薄弱点，调整内容难度
Vector search · user namespace
5
Quiz Node —— 测验生成
基于学习内容 + 错题队列生成测验（JSON）
GPT-4o-mini · conditional edge
6
Evaluator Node —— 评估与决策
正确率<60% 回补资源，≥60% 前进。错题存入 Pinecone
Conditional routing · memory write
7
流式输出到前端
SSE 实时推送，用户看到 Agent 思考过程
streamEvents() → SSE → React
